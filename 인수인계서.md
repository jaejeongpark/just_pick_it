# just_pick_it 협업 인수인계서

## 워크스페이스 구조

이 워크스페이스는 **하나의 git 저장소**로 구성되어 있습니다.

```
just_pick_it/                   ← 저장소 (github.com/jaejeongpark/just_pick_it)
├── src/
│   ├── pinky_pro/              ← Pinky Pro AMR 플랫폼 패키지
│   ├── mycobot_ros2/           ← MyCobot 로봇 패키지
│   └── just_pick_it/          ← 프로젝트 패키지 (각자 담당 패키지에서 작업)
│       ├── just_pick_it_interfaces/       ← 팀 리더 관리 (공용 msg/srv/action)
│       ├── just_pick_it_perception/       ← 팀 리더 담당
│       ├── just_pick_it_simulation/       ← 담당자 미정
│       ├── jetcobot_inspection/           ← 팀 리더 담당
│       ├── jetcobot_sorting/              ← 팀원 A 담당
│       ├── pinky_amr_1/                   ← 팀원 B 담당
│       ├── pinky_amr_2/                   ← 팀원 C 담당
│       └── just_pick_it_bringup/          ← 팀 리더 관리 (통합 launch)
├── scripts/
└── README.md
```

모든 작업은 **본인 담당 `feature/` 브랜치**에서 진행하고, `dev` 브랜치를 통해 통합합니다.

---

## 1. 최초 1회 설정

처음 이 워크스페이스를 받을 때만 실행합니다.

```bash
# 1. 워크스페이스 클론
git clone https://github.com/jaejeongpark/just_pick_it.git

# 2. 워크스페이스 폴더로 이동
cd just_pick_it

# 3. 본인 feature 브랜치로 전환
git checkout feature/<본인_브랜치명>

# 4. 의존성 설치
rosdep install --from-paths src --ignore-src -r -y

# 5. 빌드
colcon build --symlink-install

# 6. ROS2 환경 설정 (매 터미널 세션마다 필요)
source install/setup.bash
```

> **`~/.bashrc`에 추가해두면 터미널을 열 때마다 자동 적용됩니다.**
> ```bash
> source ~/just_pick_it/install/setup.bash
> ```

## 2. 매일 작업 시작 전 (최신 코드 받기)

**코드 수정 전에 항상 실행**하세요.

```bash
# 1. 원격 변경사항 가져오기
git fetch origin

# 2. 내 작업 상태 확인
git status

# 3. dev의 최신 내용을 내 브랜치에 반영
git merge origin/dev

# 4. 빌드
colcon build --symlink-install

# 5. 환경 설정
source install/setup.bash
```

### just_pick_it_interfaces가 변경된 날 (팀장이 별도 공지)

메시지·서비스·액션 정의가 바뀐 경우, 다른 패키지보다 interfaces를 먼저 빌드해야 합니다.

```bash
colcon build --packages-select just_pick_it_interfaces --symlink-install
colcon build --symlink-install
source install/setup.bash
```

## 3. 패키지 수정 절차

`src/` 하위 본인 담당 패키지를 수정했을 때의 절차입니다.
(`just_pick_it/`, `pinky_pro/`, `mycobot_ros2/` 모두 동일합니다.)

### Step 1. 워크스페이스 루트에서 시작

```bash
cd ~/just_pick_it
```

### Step 2. 내가 수정한 파일 확인

```bash
git status
```

### Step 3. 수정한 파일 스테이지에 올리기

```bash
# 특정 패키지 폴더 전체
git add src/just_pick_it/<패키지명>/

# 또는 특정 파일
git add src/just_pick_it/<패키지명>/some_file.py
```

### Step 4. 커밋하기

형식: `타입 : 내용` — 타입은 대문자, 콜론 양쪽에 공백 한 칸

```bash
git commit -m "FEAT : 변경 내용을 한 줄로 요약"
```

| 타입 | 사용 상황 |
|------|----------|
| `FEAT` | 새 기능 추가 |
| `FIX` | 버그 수정 |
| `CHORE` | 설정·빌드 변경 |
| `DOC` | 문서 수정 |
| `REFACTOR` | 기능 변경 없는 코드 정리 |
| `INIT` | 패키지·파일 최초 생성 |

예시: `git commit -m "FIX : nav2 기본 맵 경로 수정"`

### Step 5. 내 feature 브랜치에 push

```bash
git push origin feature/<본인_브랜치명>
```

## 4. just_pick_it_interfaces 수정 시 (팀장 전용)

메시지·서비스·액션 정의를 변경했을 때는 빌드 확인 후 push하고 팀원에게 공지하세요.

```bash
# 1. 인터페이스 빌드 확인
colcon build --packages-select just_pick_it_interfaces --symlink-install

# 2. 이상 없으면 커밋 후 push
git add src/just_pick_it/just_pick_it_interfaces/
git commit -m "FEAT : DetectedObject msg에 velocity 필드 추가"
git push origin feature/ai_jj
```

push 후 팀원들에게 **"interfaces 변경됨, 오늘 아침 루틴에 interfaces 빌드 추가"** 공지하세요.

## 자주 발생하는 문제 해결

### `from just_pick_it_interfaces.msg import ...` 에러가 나요

interfaces 패키지를 빌드하지 않은 상태입니다.

```bash
colcon build --packages-select just_pick_it_interfaces --symlink-install
source install/setup.bash
```

### 빌드 후 변경사항이 반영이 안 돼요

```bash
colcon build --symlink-install
source install/setup.bash
```

`source install/setup.bash`는 새 터미널을 열 때마다 실행해야 합니다.

### push 시 아래 에러가 나요

```
 ! [rejected]  feature/xxx -> feature/xxx (non-fast-forward)
error: failed to push some refs
```

다른 사람이 같은 브랜치에 먼저 push했습니다. pull 후 다시 push하세요.

```bash
git pull origin feature/<본인_브랜치명>
git push origin feature/<본인_브랜치명>
```

### merge 시 충돌(conflict)이 났어요

```bash
# 충돌 파일 확인
git status

# 충돌 파일을 편집기로 열어 직접 해결 후
git add <충돌_파일>
git commit
```
