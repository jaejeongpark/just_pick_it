# Reverse Docking — 정밀 후진 도킹 설계 (1-Slide 요약)

**마커 + 노란 주차라인 하이브리드 · 차동구동 AMR · 좁은 충전 도크 정밀 도킹**

---

### ① 문제 — 정밀도 제약을 기하로 역산

좁은 도크에 후진 삽입. 후진 0.33 m에서 횡오차 1 cm 이내 →
**`0.01 = 0.33·sin(θ)` ⇒ 핵심 제약 `yaw ≲ 1.7°`**

### ② 고찰 — 병목은 yaw

| 축 | 단일 마커 solvePnP | 판정 |
|---|---|---|
| Lateral | 1 px ≈ **0.5 mm** | 쉬움 |
| **Yaw** | **1~3° + ambiguity** | **병목 (← 제약 1.7°)** |

### ③ 선택 — 노이즈 센서는 1회만, 실행은 odom *(실차 브링업 후 개정)*

원설계는 "라인이 yaw 담당"이었으나 실차에서 **라인은 standby 거리에서 불검출**, **마커 rvec yaw는 정지 상태에서도 ±5~8° 출렁(ambiguity)** → 실시간 마커 서보가 발산. 그래서:

| 축 | 담당 (개정) | 방식 |
|---|---|---|
| **횡 위치** (법선 진입) | 마커 **1회 측정** + **odom arc** | psi 노이즈를 측정 1회로 격리, 이동은 odom |
| **헤딩 유지** (후진) | 마커로 법선 1회 + **odom yaw 앵커** | psi 대신 odom 으로 후진 내내 정밀 유지 |
| depth · 정지 | ArUco 마커 | 월드 깊이 `robot_y ≤ dock_y` (원거리 scale) |
| 위치보정 | ArUco 마커 | 도킹 후 `/initialpose` |

> **원칙**: ① 노이즈 센서(rvec/psi)는 "1회·정지·평균"으로만. ② **측정 전 똑바로 세우면**(psi≈0) `Δx≈−tx·scale` 로 깨끗. ③ odom 은 짧은 상대이동·헤딩 유지엔 가장 정밀("cmd_vel 을 믿는다"의 정확한 구현).

### ④ 검증 — 실차 결과 (2026-06-11)

| 구간 | 결과 |
|---|---|
| 마커 검출 · 1회 측정(Δx) | ✅ 실측과 ≈일치 (예: 시작 0.165 → Δx 0.053, 자 0.055) |
| odom arc 법선 진입 | ✅ 부드러운 단일 호, odom 종료판정 정확 |
| **후진 중 odom yaw 유지** | ✅ `yaw_drift` ±0.2° 고정 (헤딩 안 흔들림) |
| 도킹 성공 (E2E) | ✅ |
| 정렬 후 x · 깊이 정지 | 🔶 잔차 노브(`marker_lat_offset_m`)·`depth_scale` 미세조정 중 |

| 지표 | 목표 | 상태 |
|---|---|---|
| Lateral 오차 | ≤ 10 mm | 🔶 floor ~1cm(단일 마커 한계) |
| Yaw 오차 | ≤ 1.5° | ✅ odom 유지로 후진 중 ±0.2° |
| 도킹 성공률 (N회) | ≥ 95 % | 🔶 N회 정량화 예정 |

> **개발자 코멘트** — *"정밀도 한계를 기하로 먼저 못 박는다"는 출발점은 유효했다. 다만 실차가 가르쳐준 건 **센서 노이즈의 구조** — 마커 yaw 는 매 프레임 쓰면 독이고, 1회 측정·정지 평균으로 쓰면 약이다. 노이즈 큰 신호는 "한 번만", 정밀 실행은 "odom" 에 넘긴 게 keystone 이었다.*
