# Reverse Docking — 정밀 후진 도킹 설계 (1-Slide 요약)

**두 ArUco 마커 translation 헤딩 + odom 정밀 곡선 · 차동구동 AMR · 좁은 충전 도크 정밀 도킹**

---

### ① 문제 — 정밀도 제약을 기하로 역산

좁은 도크에 후진 삽입. 후진 0.33 m에서 횡오차 1 cm 이내 →
**`0.01 = 0.33·sin(θ)` ⇒ 핵심 제약 `yaw ≲ 1.7°`**

### ② 고찰 — 병목은 yaw

| 축 | 단일 마커 solvePnP | 판정 |
|---|---|---|
| Lateral | 1 px ≈ **0.5 mm** | 쉬움 |
| **Yaw** | **1~3° + ambiguity** | **병목 (← 제약 1.7°)** |

### ③ 선택 — 노이즈 센서는 1회만, 실행은 odom · 헤딩은 두 마커 *(실차 개정 06-11 → 06-15)*

원설계 "라인이 yaw 담당"은 실차에서 깨졌고(라인 standby 불검출, 마커 rvec yaw ±5~8° ambiguity),
끝까지 파보니 **두 근본원인**이 더 있었다:
- **카메라 fx 가 1.48배 작음** → tz·회전·두-마커 절대좌표 오염. `depth/lateral_scale` 은 그걸 덮던 fudge. (tx 는 fx 무관 → 정확)
- **단일 평면 마커론 헤딩이 구조적 병목**(pose-flip) → **두 마커 translation** 으로 ambiguity 없이 yaw 확보.

| 축 | 담당 (최종) | 방식 |
|---|---|---|
| **헤딩(yaw)** | **두 마커 translation** | `psi_two=atan2(Δtz·fx, Δtx)`, pose-flip 없음(σ±0.05°) |
| **횡·위치** | 두 마커 강체정합 → **odom 곡선 이동** | robot(x,y) 1회 측정 → (dock_x, approach_y, 법선)으로 정밀 주차 |
| depth · 정지 | ArUco 마커 | 월드 깊이 `robot_y ≤ dock_y` |
| 위치보정 | ArUco 마커 | 도킹 후 `/initialpose` |

> **원칙**: ① 노이즈 센서(단일 rvec/psi)는 안 쓰거나 "1회·정지". ② **두 기지점(마커 2개)의
> translation 으로 헤딩을 ambiguity 없이** 잡는다. ③ odom 은 짧은 상대이동·곡선·헤딩 유지엔 가장 정밀.

### ④ 검증 — 단일 마커 vs 두 마커 (실측)

| 항목 | 단일 마커 | 두 마커 |
|---|---|---|
| 헤딩 측정 노이즈 | ±3~8° (pose-flip) | **±0.05°** |
| 두 마커 yaw 자기일치 | 4° 불일치(6.9 vs 11.1°) | 자동 일치 |
| 깊이 의존 x 편향 | ±1.4cm/깊이 | **없음** |
| 최종 도킹 x 오차 | ~1cm | **~3~5mm** |
| 최종 도킹 yaw | 비뚤(2~3°) | **곧음(<1°)** |

| 지표 | 목표 | 상태 |
|---|---|---|
| Lateral 오차 | ≤ 10 mm | ✅ ~3~5mm (두 마커) |
| Yaw 오차 | ≤ 1.5° | ✅ <1° (두 마커 법선 + odom) |
| E2E 도킹 | 성공 | ✅ **완성** (좁은 도크 안정) |

> **개발자 코멘트** — *기하로 못 박은 제약(yaw≲1.7°)이 끝까지 옳았다. 단일 마커는 횡은
> 쉬웠지만 그 yaw 가 병목이었고, "추측으로 부호 튜닝" 대신 **정지 진단툴로 fx 오류와 pose-flip
> 을 숫자로 격리**한 게 keystone. 노이즈 센서는 버리고(또는 1회), **두 기지점의 translation 으로
> 헤딩을 잡고 odom 으로 정밀 실행** — 문제를 게인이 아니라 구조로 풀었다.*
