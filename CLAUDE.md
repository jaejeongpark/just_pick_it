# Workspace Rules

## XML / URDF / SDF 파일 주석 규칙

XML 계열 파일(`.urdf`, `.xacro`, `.sdf`, `.launch`, `.xml`)을 수정할 때 주석 작성에 다음 규칙을 반드시 따른다.

### 금지 사항

XML 주석(`<!-- -->`) 내부에 아래 문자열과 문자를 절대 사용하지 않는다.

- `--` (더블 대시) — XML 명세상 주석 내부에 허용되지 않아 파싱 오류 발생
- `-->` — 주석 종료 토큰으로 오인되어 구조 붕괴
- `->` — 위와 동일한 이유로 혼동 유발 가능
- `→` `←` `↑` `↓` `⇒` `⟶` 등 유니코드 화살표 문자 — 인코딩 문제 및 파서 호환성 문제 유발 경험 있음

```xml
<!-- 잘못된 예: link A --> link B 순서로 연결 -->   <!-- 파싱 오류 -->
<!-- 잘못된 예: parent--child 관계 설명 -->          <!-- 파싱 오류 -->
<!-- 잘못된 예: base → end_effector 방향 -->         <!-- 유니코드 화살표 금지 -->
```

### 올바른 작성 방법

주석은 화살표나 기호 없이 평서문으로 작성한다.

```xml
<!-- 올바른 예: link A에서 link B 순서로 연결 -->
<!-- 올바른 예: parent와 child 관계 설명 -->
<!-- 올바른 예: base_link 기준 offset은 x=0.1 y=0.0 z=0.05 -->
```

### 주석을 생략해도 되는 경우

태그 이름과 속성값만으로 의미가 명확한 경우 주석을 달지 않는다. 주석은 값의 단위, 좌표계, 비자명한 수치의 출처 등 코드만으로 알 수 없는 정보에만 추가한다.

## 커밋 메시지 규칙

### 금지 사항

커밋 메시지, PR 본문, 코드 주석, 문서 어디에도 Claude / Anthropic / AI 어시스턴트 관련 작성자(Author) 또는 공동작성자(Co-Author) 표기를 절대 추가하지 않는다.

다음과 같은 트레일러와 문구를 모두 금지한다.

- `Co-Authored-By: Claude ...`
- `Co-Authored-By: Anthropic ...`
- `Generated with Claude Code` 류의 자동 생성 안내 문구
- 그 외 Claude / Anthropic / AI / 어시스턴트가 작성/기여했음을 드러내는 모든 트레일러

커밋은 항상 실제 작성자(`git config user.name` / `user.email`) 단독 명의로만 작성한다.
